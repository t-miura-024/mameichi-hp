# 要求定義書 - 焙煎工房 函館豆壱 ホームページ

## 1. プロジェクト概要

### 1.1 背景

焙煎工房 函館豆壱は、北海道函館市に店舗を構えるコーヒー豆の専門店である。現在、Instagram および Facebook を中心に情報発信を行っているが、お店の詳細情報やこだわりを体系的に伝える自社メディアを持っていない。既存の SNS フォロワーや常連客に対してより深い情報を届けるとともに、新規顧客の獲得につなげるため、公式ホームページの開設を行う。

### 1.2 目的

本プロジェクトの目的を優先順位順に以下に示す。

| 優先度 | 目的 | 概要 |
|:---:|---|---|
| 1 | 来店促進 | 既存顧客の LTV 向上を目的とし、営業情報・新着豆・お知らせを定期的に発信することで来店頻度を高める |
| 2 | SNS フォロワーの誘導 | Instagram・Facebook のフォロワーをホームページへ誘導し、お店のこだわりや商品情報をより深く知ってもらう |
| 3 | ブランディング・認知向上 | 函館の自家焙煎コーヒー豆専門店としてのブランドを確立し、検索経由での新規認知を獲得する |
| 4 | オンライン販売（将来） | 将来的な EC 機能の追加を見据え、拡張可能な設計とする（初期リリースには含めない） |

### 1.3 スコープ

#### 初期リリースに含めるもの

- 店舗情報・アクセス情報の掲載
- お店のコンセプト・こだわりの紹介
- 商品の一覧表示
- お知らせ・ブログ機能
- お問い合わせフォーム
- SNS 連携（リンク・埋め込み表示）
- SEO 対策

#### 初期リリースに含めないもの

- オンラインショップ（EC）機能
- 会員登録・ログイン機能
- 予約機能

## 2. ビジネス要求

### 2.1 来店促進（最優先）

- 営業時間・定休日・アクセス情報をわかりやすく掲載し、来店のハードルを下げる
- 新着商品やおすすめ情報を定期的に発信し、来店動機を創出する
- 季節限定商品の告知により、リピート来店を促す

### 2.2 SNS フォロワーの誘導

- SNS プロフィールからホームページへ遷移した際に、お店の魅力が伝わるトップページを構築する
- Instagram の最新投稿を埋め込み表示し、SNS とホームページの相互回遊を促進する
- SNS では伝えきれない詳細情報（商品の一覧、焙煎のこだわり等）をホームページで補完する

### 2.3 ブランディング・認知向上

- 「函館の自家焙煎コーヒー豆専門店」としてのポジションを確立する
- 豆の仕入れ・選定、焙煎へのこだわりをストーリーとして伝える
- 函館という土地に根ざしたお店であることを前面に打ち出す
- Google 検索での上位表示を目指し、新規顧客の流入を図る

### 2.4 将来的なオンライン販売

- 初期リリース後、EC 機能を追加できるよう拡張性を確保する
- 商品データ（商品一覧）は EC 連携を見据えた構造で管理する

## 3. ターゲットユーザー

### 3.1 プライマリターゲット：常連客

- 函館市在住で、普段から豆壱を利用しているお客様
- 新着豆やお知らせを定期的にチェックし、来店の参考にする
- スマートフォンでの閲覧が主

### 3.2 セカンダリターゲット：コーヒー好きの観光客

- 函館を訪れる観光客のうち、コーヒーに興味がある層
- 「函館 コーヒー豆」「函館 自家焙煎」等のキーワードで検索して訪問する
- お店の雰囲気・アクセス情報・取扱い商品を確認して来店を判断する
- スマートフォンでの閲覧も多い

---

## 4. コンテンツ要求

### 4.1 ページ構成

| ページ | 主な目的 | 概要 |
|---|---|---|
| トップページ | 第一印象・回遊の起点 | お店の魅力を伝えるビジュアル、最新のお知らせ、SNS 埋め込み、各ページへの導線 |
| お店について | こだわりの訴求 | コンセプト、豆の仕入れ・選定のこだわり、焙煎へのこだわり、函館との関わり |
| 商品一覧 | 商品情報の提供 | 常時 20〜30 種類の商品を一覧表示。産地・焙煎度・特徴など。季節限定・おすすめの表示 |
| アクセス・店舗情報 | 来店促進 | 住所、営業時間、定休日、地図（Google Maps）、最寄り駅からの案内 |
| お知らせ | 最新情報の発信 | 新着豆、イベント、営業日変更等の告知。一覧表示＋詳細ページ |
| お問い合わせ | 顧客接点の確保 | 名前・メールアドレス・お問い合わせ内容のシンプルなフォーム |

### 4.2 共通要素

- ヘッダー：ロゴ、ナビゲーション
- フッター：店舗情報（住所・電話番号・営業時間）、SNS リンク、著作権表示

### 4.3 CMS で管理するコンテンツ

以下のコンテンツは店主がスマートフォンから随時更新できるようにする。

- お知らせ記事
- 商品情報

## 5. デザイン要求

### 5.1 カラー

- **ベース**: モノトーン（白・グレー・黒）
- **アクセント**: コーヒーブラウン系

### 5.2 雰囲気

- 庶民的で親しみやすさがありつつも、落ち着いた洗練された印象
- 豆売り専門店としての専門性・信頼感を感じさせるデザイン
- 函館という土地の空気感を感じさせる要素

### 5.3 参考サイト

| サイト名 | 参考にするポイント |
|---|---|
| [Chevron Coffee Roasters](https://chevroncoffee.jp/) | シンプルで洗練されたレイアウト、ロースタリーとしての世界観 |
| [びっくりドンキー コーヒー LP](https://www.bikkuri-donkey.com/lp/2024_coffee/) | ストーリー性のあるコンテンツの見せ方 |
| [PostCoffee](https://postcoffee.co/) | 商品（豆）の一覧表示、カテゴリ分けの参考 |
| [ハニー珈琲](http://www.honeycoffee.co.jp/) | コーヒー関連事業のコーポレートサイトとしての構成 |

### 5.4 ロゴ・写真素材

- 現時点ではロゴ・写真素材ともに未準備
- 開発と並行して用意する、もしくはプレースホルダーで対応し後から差し替える

## 6. 運用要求

### 6.1 更新体制

- **更新者**: 店主（開発者ではない非エンジニア）
- **更新デバイス**: スマートフォン
- **更新頻度**: 毎営業日（月・木・金・土・日）
- **更新内容**: お知らせ投稿、商品情報の追加・変更

### 6.2 更新の容易さ

- SNS 投稿と同程度の手軽さで、テキストと画像を入力・公開できること
- 専門的な知識（HTML、プログラミング等）を必要としないこと
- microCMS の管理画面を利用し、直感的な操作で完結すること

### 6.3 デプロイ

- コンテンツ更新時は CMS の Webhook により自動でビルド・デプロイされること
- コード変更時は GitHub への push により自動でビルド・デプロイされること
